# Triage Labels

The skills speak in terms of five canonical triage roles. This file maps those roles to the actual label strings used in this repo's issue tracker.

| Label in mattpocock/skills | Label in our tracker | Meaning |
| --- | --- | --- |
| `needs-triage` | `needs-triage` | Maintainer needs to evaluate this issue or PR |
| `needs-info` | `needs-info` | Waiting on reporter or PR author for more information |
| `ready-for-agent` | `ready-for-agent` | Fully specified, ready for an agent |
| `ready-for-human` | `ready-for-human` | Needs owner judgment, taste, or approval |
| `wontfix` | `wontfix` | Will not be actioned |

When a skill mentions a role, use the corresponding label string from this table.

## Optional Area Labels

Add these only when useful:

- `area:course`
- `area:forum`
- `area:auth`
- `area:deployment`
- `area:design`
- `area:student-pr`

## Triage Notes

Use `ready-for-agent` only when the expected output is concrete enough to verify.

Use `ready-for-human` for decisions involving ideology, tone, paid content boundaries, visual taste, community norms, or final editorial judgment.
