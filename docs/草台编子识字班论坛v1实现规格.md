# 草台编子识字班 · 论坛 v1 实现规格

配套决策：[docs/adr/2026-06-23-forum-and-agent-access-model.md](adr/2026-06-23-forum-and-agent-access-model.md)。
本规格是可演进的实现文档，不是 ADR；落地细节以本文件为准，决策边界以 ADR 为准。

## 1. 范围

做（第一版只做课程社区需要的）：

- 帖子列表、发帖、回帖、标签、作业分享标签。
- 管理员置顶、管理员隐藏/删除（软删除）。
- 作者编辑自己的帖子/回复。
- 未解锁用户不可访问。

不做：私信、复杂通知、等级勋章、富文本编辑器、大型附件系统。

## 2. 路由

```txt
/forum              帖子列表（支持置顶在前）
/forum/new          发帖
/forum/t/[slug]     帖子详情 + 回复
/forum/tag/[tag]    按标签筛选
```

- 全部 `/forum/**` 需要登录且持有有效 entitlement；无权限渲染解锁引导（复用课程页 `Gate` 的同款形态），不泄漏正文。
- 详情/列表为 Server Component；发帖、回帖、编辑、管理为 Server Action。

## 3. 数据

复用既有表（`db/schema.ts`，不新增表）：`forum_posts` / `forum_comments` / `forum_tags` / `forum_post_tags`。

- `status` 三态：`published | hidden | deleted`，一律软删除，不硬删。
- `pinned_at` 非空表示置顶；列表排序：置顶（`pinned_at` 倒序）→ `created_at` 倒序。
- `forum_posts.slug` 唯一：用 `短随机 id + 标题 slug 化` 生成（例：`a1b2-zen-and-the-art`），避免中文标题碰撞与目录穿越；slug 仅 `[a-z0-9-]`。
- 作业分享 = 内建标签（slug 固定，如 `homework`），不另起系统。
- 管理员可新增、改名、隐藏/恢复标签；标签 slug 创建后不改，隐藏标签不再给学员选择，也不在公开帖子标签中展示。

校验上限（服务层强制）：标题 ≤ 120 字，正文 ≤ 20000 字，单帖标签 ≤ 5。

## 4. 服务层 `lib/forum.ts`（本规格核心）

所有读写都经此层；session、entitlement、作者/管理员授权、限流、软删除只在这里。UI 现在调它，未来 MCP / REST 调同一批函数。沿用 `lib/content.ts` 的入参风格（`env` + `requestHeaders`）。

```ts
// 已鉴权的访问者上下文：在服务层内部由 session + profiles + entitlement 解析得出
type Viewer = {
  userId: string;
  displayName: string;
  role: "student" | "admin";
  hasForumAccess: boolean; // v1 即"持有 course:full"
};

// 解析访问者；无 session 返回 null
resolveViewer({ env, requestHeaders }): Promise<Viewer | null>

// 读：无权限返回 null（页面渲染 Gate），不抛
listPosts({ env, requestHeaders, tag?, cursor?, limit? }): Promise<PostListPage | null>
getThread({ env, requestHeaders, slug }): Promise<ThreadView | null>
listTags({ env }): Promise<Tag[]>

// 写：返回判别式结果，区分 forbidden / rate_limited / invalid / ok
createPost({ env, requestHeaders, input: { title, bodyMd, tagSlugs } }): Promise<WriteResult<{ slug }>>
addComment({ env, requestHeaders, postId, bodyMd }): Promise<WriteResult<{ id }>>
editPost({ env, requestHeaders, postId, patch }): Promise<WriteResult>   // 作者本人或管理员
editComment({ env, requestHeaders, commentId, bodyMd }): Promise<WriteResult>

// 管理：仅 role=admin
moderatePost({ env, requestHeaders, postId, action: "pin" | "unpin" | "hide" | "delete" }): Promise<WriteResult>
```

约定：

- 读函数对"无权限"与"不存在"都返回 `null`，调用方一视同仁渲染 Gate / 404，不区分以免探测。
- 写函数返回 `WriteResult`（`{ ok: true, ... } | { ok: false, reason }`），Server Action 据此回显错误。
- entitlement 判定复用 `lib/content.ts` 同款查询（`scope = course:full`，`starts_at <= now`，`expires_at` 空或未过期）；scope→能力映射封装在本层，便于日后拆 `forum:*`。

### 4.1 第 1 轮落地的 payload 类型（前端反向定稿，已冻结）

第 1 轮按 UI 需要把上面读函数的返回体定了型，落在 `lib/forum.ts`。第 2 轮真实实现照此返回，**字段不减**（按需可加）：

- `Tag = { slug, name }`；`Author = { userId, displayName, qq: string | null }`（无 QQ → 默认头像）。
- `ContentStatus = "published" | "hidden" | "deleted"`。
- `PostSummary`（列表项，不含正文）：`{ id, slug, title, author, tags, status, pinned, createdAt, commentCount, excerpt }`。`excerpt` 由服务层从正文裁出纯文本，列表不接触全文。
- `PostListPage = { posts: PostSummary[], nextCursor: string | null, tag: Tag | null, viewer }`。
- `ThreadView = { post, comments, viewer }`，其中 `post` 含 `bodyMd` 与 **`canEdit`（作者或管理员）/ `canModerate`（管理员）**；`comments[]` 每条含 `canEdit`。授权布尔由服务层算好给 UI，UI 不自行判断 role。
- 写函数 `WriteResult` 的失败分支补 **`fieldErrors?: { title?, bodyMd?, tagSlugs? }`**（`reason: "invalid"` 时逐字段回显）与可选 `message`；`reason` 闭集仍是 `forbidden | rate_limited | invalid`。
- 上限常量从本层导出：`TITLE_MAX=120`、`BODY_MAX=20000`、`MAX_TAGS=5`，前后端与表单计字共用一处。

## 5. 授权矩阵

| 能力 | 要求 |
| --- | --- |
| 读论坛 | 登录 + 有效 entitlement |
| 发帖 / 回复 | 登录 + 有效 entitlement |
| 编辑自己的帖/回复 | 作者本人 |
| 编辑/删除任意帖、置顶 | `profiles.role = admin` |
| 新增 / 改名 / 隐藏标签 | `profiles.role = admin` |

## 6. 安全

- **用户 Markdown 不开原始 HTML**：论坛正文渲染走 `react-markdown` + `remark-gfm`，**不引入 `rehype-raw`**，杜绝存储型 XSS（与课程正文渲染区分；课程是可信内容）。链接做协议白名单（http/https）。
- 发帖、回帖**限流**（按 userId + 时间窗，先用 D1 计数或近似实现）。
- 软删除：`hidden`/`deleted` 不在公开列表/详情渲染正文；管理员视图可见占位。
- slug、tag、分页游标全部按 `[a-z0-9-]` / 数值校验，防注入与穿越。

## 7. 复用

- 头像：`/api/avatar/qq/:qq`（已存在）+ `profiles.displayName`，无 QQ 用默认头像。
- 解锁引导：复用课程页 `Gate` 形态。
- 设计语言：沿用"暖纸墨绿 + 红点缀"既有 token 与 `print-block` / `kicker` 等组件类。

## 8. 执行节奏与落地顺序

按 [agents/collaboration.md](agents/collaboration.md) 的"前后端分轮"：本规格分**两个 agent 轮次、前端先行**，以第 4 节 `lib/forum.ts` 的类型签名为**冻结契约**对接。

当前状态（2026-06-23）：第 2 轮后端已落地。`lib/forum.ts` 已替换为真实 session + profiles + `course:full` entitlement + D1 读写实现；`FORUM_ROUTES_ENABLED` 临时护栏已拆，顶栏「论坛」入口已恢复。D1 默认标签由 `drizzle/migrations/0002_seed_forum_tags.sql` 写入，真实首帖 / 公告内容仍需通过受控 D1 写入，不进公开仓库。

**第 1 轮 · 前端（对着 typed stub）**

- 以第 4 节接口的 TypeScript 类型 + 函数签名为准，写一个返回 fixture 假数据的 stub 实现。
- 做齐路由与组件：`/forum`、`/forum/t/[slug]`、`/forum/new`、`/forum/tag/[tag]` + Gate + 题记 / 头像 / Markdown 渲染（**不开 `rehype-raw`**）。
- 目标：UI 与交互定稿，并反向定稿契约——UI 需要而契约缺的字段，改第 4 节而非将就。
- 护栏（已落地）：**生产不挂出 `/forum`**，直到第 2 轮门禁落地，避免未鉴权占位对外。两道闸：
  1. `lib/forum.ts` 导出 `FORUM_ROUTES_ENABLED = process.env.NODE_ENV !== "production"`，所有 `/forum/**` 路由与写 Action 在生产 `notFound()` / 拒绝；`next dev` 放行做 UI 验收。
  2. `components/site-header.tsx` 的 NAV 暂时移除「论坛」项。
- 第 1 轮的 `resolveViewer` 是固定 fixture viewer（`STUB_VIEWER`，含 `hasForumAccess` / `role` 开关，改一行即可验收 Gate / 学员视角）；stub 用挂在 `globalThis` 的内存仓，`next dev` 内可真增删改查，重启即重置。

**第 2 轮 · 后端（实现契约）**

- 实现真 `lib/forum.ts`：`Viewer` 解析、entitlement 门禁、D1 查询、Server Action、限流、slug 生成、软删除。
- 用真实现替换 stub；契约不变则前端不需改。最后才打开 `/forum` 入口：删掉 `FORUM_ROUTES_ENABLED` 这道闸（或改成真鉴权），并把「论坛」加回 `site-header.tsx` 的 NAV。

细分步骤（落在两轮内）：读路径（服务层读 + 读路由 + Gate）→ 写（发帖 / 回帖 Action + 校验 + 限流 + slug）→ 管理（置顶 / 隐藏 / 删除）→ 作业标签与列表筛选打磨。

## 9. 明确留到后续（不在 v1）

- MCP 适配器（复用本服务层，按用户 scoped token 鉴权，读优先）——见 ADR 第 4 条。
- 通知、订阅、@提醒、富文本、附件。
- 独立的 `forum:read` / `forum:write` entitlement scope（届时只改服务层映射）。
